### **2. Form Validation with TypeScript**

**Goal:** Implement client-side form validation for a registration form using TypeScript, providing real-time feedback.

**Key Concepts:**
*   **Form Handling:** Capturing user input and managing form submission.
*   **Input Validation:** Checking user input against predefined rules (e.g., format, length).
*   **Dynamic Error Messages:** Displaying validation feedback directly on the page.

**TypeScript Specifics:**
*   **Type Definition (`FormData`):**
    *   Defines the structure of the data expected from the form fields: `username` (string), `email` (string), `password` (string).
*   **Validation Functions:**
    *   Separate functions for each field's validation (e.g., `validateUsername`, `validateEmail`, `validatePassword`).
    *   These functions should return `true` for valid input and `false` or an error message for invalid input.
    *   **Email Format:** Use a regular expression (regex) to check for a valid email pattern.
    *   **Password Length:** Check if the password string meets a minimum length requirement.
*   **Event Handling:** Attaching event listeners (e.g., `submit` for the form, `input` or `change` for individual fields) to trigger validation.

**HTML & CSS Role:**
*   **HTML:**
    *   Design a `<form>` with `<label>` and `<input>` elements for username, email, and password.
    *   Include `<span class="error-message"></span>` elements next to each input field to display specific error messages.
    *   Add a `<button type="submit">Register</button>`.
    *   Include a `div` or `p` element for a global success message.
*   **CSS:** Style the form, input fields, and especially the error messages (e.g., red text for errors, highlighting invalid input fields with a red border).

**Implementation Steps:**
1.  **HTML Form:**
    *   Create `index.html` with the registration form structure, including placeholders for error messages and a success message.
2.  **TypeScript Event Listeners:**
    *   Get references to the form and all input fields.
    *   Attach a `submit` event listener to the form.
    *   (Optional but good practice) Attach `input` or `blur` event listeners to individual fields for real-time validation.
3.  **Validation Logic:**
    *   In the `submit` event handler:
        *   Prevent the default form submission (`event.preventDefault()`).
        *   Retrieve values from input fields.
        *   Call your validation functions for each field.
        *   Maintain a `isValid` flag.
        *   If a field is invalid:
            *   Set `isValid` to `false`.
            *   Display the specific error message in the corresponding `span`.
            *   Add an "invalid" class to the input field.
        *   If valid:
            *   Clear any error messages and remove "invalid" class.
    *   If all fields are `isValid` after checking:
        *   Display the success message.
        *   (Optional) Clear the form fields.
        *   (Optional) Simulate data submission (e.g., `console.log(formData)`).
4.  **Styling:** Use CSS to enhance the form's appearance and clearly differentiate error states.

**TypeScript Specifics:**

*   **Async Function (`fetchData`):**
    *   Marked with the `async` keyword.
    *   Uses `await` to pause execution until a Promise resolves (simulated API call).
*   **Promise Simulation:**
    *   Use `new Promise()` or `Promise.resolve()` combined with `setTimeout()` to simulate a network delay and returning data.

**HTML & CSS Role:**
*   **HTML:**
    *   A `<div id="loading-message">Loading data...</div>` initially visible.
    *   A `<div id="data-container"></div>` to display the fetched content, initially hidden.
*   **CSS:** Style the loading message and the dynamically rendered data for a clear, organized layout.

**Implementation Steps:**
1.  **HTML Structure:**
    *   Create `index.html` with the loading message and data container.
2.  **TypeScript Interface:**
    *   Define the `Post` interface.
3.  **Simulate API Fetch:**
    *   Create an `async function fetchPosts(): Promise<Post[]>`:
        *   Show the `loading-message` div.
        *   Use `await new Promise(resolve => setTimeout(resolve, 2000));` to simulate a 2-second network delay.
        *   Define an array of sample `Post` objects to be "fetched".
        *   Return this array.
4.  **Render Data Function:**
    *   Write a function (e.g., `renderPosts(posts: Post[])`) that:
        *   Gets the `data-container`.
        *   Clears its content.
        *   Iterates through the `posts` array.
        *   Creates and appends HTML elements (e.g., `div` for a post, `h3` for title, `p` for body).
5.  **Main Async Logic:**
    *   In an `async` block or immediately invoked `async` function:
        *   Call `const posts = await fetchPosts();`.
        *   Hide the `loading-message` div.
        *   Call `renderPosts(posts);`.
        *   Show the `data-container` div.
6.  **Styling:** Apply CSS to the loading state and the rendered posts to make them visually distinct and appealing.
